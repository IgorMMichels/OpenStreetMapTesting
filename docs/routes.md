# Routes Documentation

This document lists and explains the application's routes, their purposes, and associated components.

Note: Replace with actual route configurations from your router setup.

- /            - Home page
- /about       - About/Info page
- /projects    - Projects dashboard
- /settings    - User and app settings
- /auth        - Authentication screens (login, signup, reset)
- /profile     - User profile and preferences

Routing Notes:
- All routes are defined in src/router/index.js (or equivalent).
- Route guards and authentication handling should be implemented per route as needed.
- Consider code-splitting and lazy loading for larger routes.
